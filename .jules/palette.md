## 2024-05-14 - Accessible Ad-hoc Icon-Only Buttons
**Learning:** In the `apps/console` UI, there is a pattern of using ad-hoc inline `<button>` elements containing only an icon (e.g., the `Eye`/`EyeOff` buttons for toggling visibility of secrets). These buttons lacked critical accessibility features: they didn't have accessible names (`aria-label`) or tooltip hints (`title`), and their custom styling inadvertently missed clear focus indicators for keyboard users.
**Action:** When creating or modifying standalone icon buttons that do not use a standard system UI component (like the `Button` component in `system.tsx`), explicitly ensure `aria-label`, `title`, and clear focus styles (like `focus-visible:ring-2 focus-visible:outline-none focus-visible:ring-white/20`) are applied so screen readers and keyboard users can effectively interact with them.
