## 2024-05-27 - Loading state implementation differs across workspaces
**Learning:** The `console` app implements an elegant built-in `busy={true}` prop on its Button component which automatically handles the loading spinner. The `web` app uses standard shadcn-style Buttons which lack this internal state, leading to inconsistent async UX where forms freeze without visual feedback.
**Action:** When working in `apps/web`, always manually compose `<Loader2 className="mr-2 h-4 w-4 animate-spin" />` inside `<Button>` for async forms to match the perceived performance of the console app.
## 2026-06-08 - Dynamic aria-describedby for Field component
**Learning:** The `Field` component in `apps/console/src/components/console/system.tsx` wraps inputs with a visually associated `hint` text. However, this hint was not programmatically associated with the input element, leaving screen reader users without important context when focusing the field. We can fix this by generating a unique ID for the hint and mapping it dynamically to the child element's `aria-describedby` prop using `React.cloneElement`, ensuring we preserve any existing `aria-describedby` values the child might already have.
**Action:** When creating wrapper components that provide descriptive text or hints to inputs, always dynamically map an ID to `aria-describedby` on the input child to ensure screen reader compatibility, combining it safely with existing props.
