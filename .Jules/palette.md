## 2024-05-27 - Loading state implementation differs across workspaces
**Learning:** The `console` app implements an elegant built-in `busy={true}` prop on its Button component which automatically handles the loading spinner. The `web` app uses standard shadcn-style Buttons which lack this internal state, leading to inconsistent async UX where forms freeze without visual feedback.
**Action:** When working in `apps/web`, always manually compose `<Loader2 className="mr-2 h-4 w-4 animate-spin" />` inside `<Button>` for async forms to match the perceived performance of the console app.
## 2026-06-04 - Enforce explicit label associations in React wrapper components\n**Learning:** When using React wrapper components (like `Field`) that wrap an input or other form element, if an explicit `id` is not provided to the child, screen readers may not associate the wrapper's label or hint with the input. Simply placing the `id` on the child isn't enough if the child has its own `aria-describedby` attributes.\n**Action:** Use `useId()` to generate a dynamic ID and `React.cloneElement` to inject it into the child input, mapping it to `<label htmlFor={id}>`. Additionally, generate an `id` for adjacent descriptive text (e.g., `hint`) and dynamically append it to the child's `aria-describedby` prop (preserving existing values) to guarantee screen reader accessibility without requiring manual assignment.
